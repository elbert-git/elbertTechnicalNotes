Test Driven Development

TDD (Test Driven Development)

A paradigm/workflow/mindest for clean and reliable code.

Red → Green → Refactor

The mantra of test-driven development. write a test. Write the code that passes the test. Then refactor. This makes sure you think deeply about and write down how the code works first before writing code. And having the tests be the by-product of this process

Important tools 

  • Jest : Javascript testing framework
  • Cypress: End to End UI/Back-end testing framework.
  • Selenium
  • RemixIDE unit Testing
